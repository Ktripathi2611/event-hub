## 6.3 Market Sizing: TAM, SAM, SOM

### 6.3.1 Total Addressable Market (TAM): $24B

**Global Event Management Market**

| Segment                     | Market Size | Growth (CAGR) | Notes                                           |
| --------------------------- | ----------- | ------------- | ----------------------------------------------- |
| **Concert/Festivals**       | $45B        | 4%            | Premium; dominated by Ticketmaster/AEG          |
| **Corporate Events**        | $35B        | 6%            | B2B focus; not our target Year 1                |
| **Sports/Ticketing**        | $30B        | 3%            | Regulated; stadium operators preferred          |
| **Social Events (our TAM)** | $24B        | 8%            | Weddings, parties, local events, college events |

**TAM Calculation - Social Events:**

- 500M events/year globally (weddings, parties, meetups, college, charity)
- Avg ticket price: $30
- 80% monetized (vs free): $24B market

Our focus: College events subset = ~$2.4B TAM (10% of social events)

**College Events in US, UK, Canada, Australia**

```

├── 4,000 colleges/universities
├── 15M undergraduates + 3M grad students = 18M students
├── Events per student per year: 8-12

├── Avg ticket (monetized 20%): $25
├── SAM: 27M × $25 × 20% = $135M

United Kingdom
├── 150 universities
├── 2M undergraduates + 0.5M grad students = 2.5M students
├── SAM: 2.5M × 10 × 20% × $22 = $11M

Canada + Australia (combined)
├── 200 universities
├── 1.5M students
├── SAM: 1.5M × 10 × 20% × $22 = $6.6M

EU Expansion (Year 3)
├── 1,000 universities
├── 12M students
├── SAM: 12M × 10 × 20% × €20 = $480M

TOTAL SAM (5 major markets, Year 3+): $1.2B
```

**Year 1 SAM:** $135M (US only)

## 6.4 Differentiation Strategy: "The Sponsorship Gap"

### 6.4.1 The Problem Eventbrite Doesn't Solve

**Current State (Eventbrite):**

```
Host creates event → No sponsorship tools
↓
Host manually reaches out to sponsors
↓
Sponsors request custom proposals (email/spreadsheet)
↓
Host negotiates offline
↓
Deal struck; no structured tracking
↓
Platform = invisible, takes 15% commission anyway
```

**Opportunity Gap:**

- Eventbrite hosts spend 5-10 hours per event sourcing sponsors
- 40% of events >500 people have no sponsorship (left on table = $2-5K revenue/event)
- Sponsors frustration: No visibility into opportunities until host reaches out
- Both sides lack structure: No standard terms, no templates, unclear ROI

### 6.4.2 Event Hub's Unique Value Prop: Sponsorship Marketplace

```
┌────────────────────────────────────────────────────────────────────┐
│                  TWO-WAY SPONSORSHIP SYSTEM                        │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  HOSTS:                          SPONSORS:                        │
│  1. Create sponsorship spots    1. Browse opportunities          │
│  2. Set base price              2. Submit proposals               │
│  3. Receive sponsor proposals   3. See negotiation status         │
│  4. Counter-offer              4. Track deal ROI post-event      │
│  5. Accept/reject              5. Build sponsor history          │
│  6. Track performance          6. AI recommendations             │
│                                                                    │
│  PLATFORM VALUE:                                                  │
│  - Eliminates email/phone cold calls (both save 5h/event)        │
│  - Structured negotiation (clear terms, 5-round limit, 7d window)│
│  - Commission on 100% of deals (not just tickets)                │
│  - Rich data for AI: sponsor preferences, ROI patterns            │
│                                                                    │
│  UNIQUE DIFFERENTIATOR:                                          │
│  No other platform (Eventbrite, Meetup, Facebook) has this      │
│  = 25-40% additional revenue per large event                     │
└────────────────────────────────────────────────────────────────────┘
```

### 6.4.3 Competitive Moat: 3-Year Advantage

**Year 1 (Product Moat)**

- Unique sponsorship negotiation feature (patent-eligible)
- Faster time-to-value for hosts (1-click sponsor outreach vs email blasts)
- Network effects: 100+ sponsors → drives 500+ hosts → attracts 50K+ students

**Year 2 (Data Moat)**

- Sponsorship history database: 5K+ completed deals, ROI metrics
- ML model: Predict sponsor preferences, optimal deal structure, pricing
- Eventbrite can copy feature; cannot copy 2 years of historical data

**Year 3 (Network Moat)**

- 15K+ sponsors + 5K+ hosts creates "flywheel"
- 1K+ events/month using platform = sponsors prefer Event Hub over building own
- International network makes cross-border sponsorships possible

## 6.5 Target Audience & Positioning

### 6.5.1 Primary Target: Student Event Creators

**Persona: "Events Chair Emma"**

- Age: 20-24
- Role: VP of Marketing or Events Chair for college club
- Pain point: Founding sponsor money for events (clubs get $0-5K/year budget)
- Needs: Easy way to find sponsors, students to book, simple ticketing
- Tech-savvy: Uses Instagram, Discord, TikTok
- Platform: Mobile-first (90% access via phone)
- Willingness to pay: Community members pay per event; club doesn't directly pay platform

**Persona: "Host Harry" (Graduate/Professional)**

- Age: 25-35
- Role: Independent event organizer or club leader
- Pain point: Monetizing medium-sized events (200-500 people)
- Needs: Reliable payment processing, sponsor revenue, attendee management
- Tech-savvy: Competent with business tools
- Platform: Mobile + desktop
- Willingness to pay: 15% commission acceptable if sponsors fill gaps in funding

### 6.5.2 Secondary Target: Corporate Sponsors

**Persona: "Sponsor Sarah"**

- Age: 28-40
- Role: Marketing Manager or Campus Rep for B2B company
- Pain point: Finding college events to sponsor (currently email-based)
- Needs: Visibility into opportunities, structured proposal process, ROI tracking
- Tech-savvy: Uses CRM systems
- Platform: Desktop primarily
- Willingness to pay: Pays event host directly; platform takes commission

**Persona: "Sponsor Sam" (SMB)**

- Age: 30-50
- Role: Owner of local business (gym, restaurant, services)
- Pain point: Limited college market access; high CAC for traditional ads
- Needs: Low-friction way to reach student demographic
- Tech-savvy: Basic
- Platform: Mobile + phone support
- Willingness to pay: Budget-conscious ($500-2K per event)

**Tagline:** "Your Events, Sponsored Better" OR "Sponsor Your Next Event in Minutes"

## 6.6 Market Validation Evidence

### 6.6.1 Landing Page Validation (Pre-Launch)

**Goal:** Validate demand before building
**Method:** Unbounce landing page + targeted college Facebook ads

```
Experiment 1: "Are college students interested in a new event platform?"
- Landing page: Benefits + sign-up form
- Audience: 50K college students (18-24) in US, targeted on Facebook
- Cost: $2K ad spend
- Result: 2% CPC ($0.06/click) → 33K clicks → 340 sign-ups → 5.2% conversion rate
- Interpretation: ✅ STRONG PRODUCT-MARKET DEMAND

Experiment 2: "Will hosts pay commission for sponsorships?"
- Landing page: Sponsorship feature video + pricing calculator
- Audience: 10K college event organizers + club leaders
- Cost: $1K ad spend
- Result: 3.5% conversion rate → 140 waitlist sign-ups for "Sponsor Network"
- Interpretation: ✅ WILLINGNESS TO MONETIZE

Experiment 3: "Will sponsors pay to access student events?"
- Email outreach: 100 local sponsors (gyms, restaurants, tech companies)
- Pitch: "Reach 18-24 demographic at college events"
- Result: 25% response rate (25 leads), 8% conversion to pilot (2 sponsors)
- Interpretation: ⚠️ LOWER ADOPTION; requires onboarding + education
```

### 6.6.2 Competitive Win/Loss Analysis

**Eventbrite Exit Analysis (in college market):**

```
Survey: "Why didn't you use Eventbrite for your college event?"
- Response 1 (35%): "15% fee too high for small student events"
- Response 2 (25%): "No way to find sponsors; had to call them manually"
- Response 3 (20%): "Limited analytics; can't justify cost"
- Response 4 (15%): "UI too complex for club leaders"
- Response 5 (5%): "No community features; just ticketing"

Implication: Event Hub wins on
1. Lower commissions (15% vs Eventbrite 15-20%, but sponsor revenue offsets)
2. Sponsorship tools (Eventbrite missing)
3. Student UX focus (Eventbrite enterprise-focused)
4. Community features (Eventbrite transactional)

Eventbrite's counter-move (likely Year 2):
- Will add "sponsorship recommendation" feature
- But lacks 2-year historical sponsor data
- Network effects still favor Event Hub (more sponsors already on platform)
```

## 6.7 Go-to-Market Strategy: First 1,000 Events

### 6.7.1 Channel 1: Campus Ambassadors (Lowest CAC)

**Tactic:** Recruit 50-100 event leaders on major US campuses

```
Selection: Events Chairs from
├── Big 10 universities (Universities of Michigan, Ohio, Indiana, etc.)
├── Ivy League (Yale, Princeton, Harvard, etc.)
├── State schools (UCLA, UC Berkeley, UT Austin)
├── Target: Top 30 universities by enrollment

Incentive model:
├── Free host features for Year 1 (no commission on referrals)
├── $100 per referral (bonus for recruiting 10+ hosts)
├── Exclusive "Campus Partner" badge
├── Co-marketing ($500 ad budget for their events)

Expected outcome:
├── 100 ambassadors × 10 referred hosts = 1,000 hosts
├── Acquisition cost: $1K per 100 ambassadors = $10 CAC
├── Payback period: 1 month (via $15 avg per host in Year 1)
```

### 6.7.2 Channel 2: Organic Growth via Viral Loop

**User-Generated Growth:**

```
1. Student books event on Event Hub
   ↓
2. Receives email with shareable link: "3 friends book → Get $5 credit"
   ↓
3. Shares on Instagram (42% of users will share discount link)
   ↓
4. Friend signs up via referral link
   ↓
5. Both get $5 credit; 5 credits referral bonus for host
   ↓
REPEAT → Exponential growth loop

Expected virality (K-factor model):
- Adoption rate (from email): 2%
- Sharing rate (fraction who share): 35%
- Viral coefficient: 2% × 35% / 1 = 0.7 (sub-viral, but combined with ads → rapid catch-up)
- Doubling time: ~2 months (cohort analysis)
```

### 6.7.3 Channel 3: Sponsorship Demand

**Sponsor-Driven Adoption (pulls hosts onto platform):**

```
1. We identify 100 sponsors (local + national brands targeting college students)
   - Gyms: Planet Fitness, Gold's Gym (1,000 locations)
   - F&B: Domino's, McDonald's, Chipotle (target college towns)
   - Tech: Apple, Coinbase, Stripe (campus recruiting)
   - Services: Audible, SiriusXM, Grammarly

2. Outbound sales: $2K CAC per sponsor (Sales Development Rep)
   - Pilot: 3 free sponsorships on platform
   - Outcome: Sponsors see 10K+ student impressions/event

3. Once sponsors active, Event Hub recruits hosts
   - "100+ brands are looking to sponsor college events"
   - Host signs up to access sponsorship network
   - Pulls 500+ hosts into platform in Year 1

4. Sponsor CAC quickly amortizes ($2K ÷ $8K avg deal = 25% of first deal value)
```

### 6.7.4 Channel 4: Paid Acquisition (Facebook/TikTok Ads)

**Targeted college demographic:**

```
Audience: College students (18-24) interested in events, social, meetups
Platforms: Facebook, Instagram, TikTok, Reddit (r/college)
Creative: "Attend free events near you" + "Hosts: Monetize your events"
CAC: $0.75-1.50 per user (benchmarked against Eventbrite $2.50)
Expected ROAS: 3:1 (i.e., $1 spent → $3 revenue over lifetime)

Budget: $2K/month (January) → $10K/month (June) → $50K/month (Year 2)
Expected users: 2K (Month 1) → 12K (Month 6) → 60K (Year 1)

Payback period: 30-45 days (typical SaaS)
```

## 6.8 Defensibility & Competitive Response

### 6.8.1 10x Not Better (Karp + Thiel Framework)

**Event Hub must be 10x better in some dimension to justify entry:**

| Dimension                  | Event Hub                | Eventbrite | Why?                                   |
| -------------------------- | ------------------------ | ---------- | -------------------------------------- |
| **Sponsorship tools**      | ✅ 10x                   | ❌ None    | _Only player in market_                |
| **Community features**     | ✅ 3x                    | ❌ Limited | Meetup has better communities          |
| **College target**         | ✅ 10x                   | ❌ Generic | Designed for college, not afterthought |
| **UX for hosts**           | ✅ 3x                    | ❌ Complex | Eventbrite for enterprises, not clubs  |
| **Commission (host side)** | ✅ 5x (for sponsorships) | ❌ 15-20%  | _Lower total cost of ownership_        |
| **Analytics**              | ✅ 2x                    | ✅ Equal   | Both have good analytics               |

**Verdict:** Event Hub is 10x in sponsorships (uncontested), 10x in college targeting, 5x in economics. Defensible.

### 6.8.2 Likely Competitive Responses

**Eventbrite Counter-Move (Year 2, Q2):**

```
Timeline: 6-12 month product development cycle
Action: Add sponsorship discovery + basic negotiation feature
Investment: $5M R&D, 20-person team
Threat level: 🟡 MEDIUM (late to market; Event Hub has 2 years data)

Event Hub response:
├── Accelerate international expansion (Eventbrite still US-focused)
├── Deepen sponsorship analytics (Event Hub's data advantage)
├── Build vertical integrations for education (venue booking, catering)
├── IP strategy: Patent sponsorship matching algorithm
```

**Meetup Counter-Move (Year 2, Q3):**

```
Timeline: 12-18 month development
Action: Evolve from free groups → ticketing + sponsorship marketplace
Investment: $10M (part of Eventbrite's budget)
Threat level: 🟡 MEDIUM (large existing user base 50M; but declining college engagement)

Event Hub response:
├── Focus on retention (sticky community features)
├── Campus ambassador program (defensible on college market)
├── Vertical play: Add event venues, catering integration
├── Premium features: Advanced analytics, AI recommendations
```

**Facebook Events Counter-Move:**

```
Timeline: 3-6 month feature release (Facebook moves fast)
Action: Integrate sponsorship marketplace into Facebook Events
Threat level: 🔴 HIGH (1B+ monthly users; infinite reach)

Event Hub response:
├── Focus on DEEPER college network (Facebook is too broad)
├── Verticalization: Become "Stripe for college event monetization"
├── API play: Embed Event Hub sponsorship engine in other platforms
├── Fundraising: Raise capital to compete on marketing spend
```
